1. Передача по значению
То, что ты делал всегда. Функция получает копию твоей переменной.

    Как выглядит: void foo(int val)

    Что происходит: Внутри функции создается новая локальная переменная val, в которую копируется значение из вызова. Это как ксерокопия документа. Что бы ты ни делал с копией внутри функции — рисовал, рвал, жег — оригинал останется нетронутым.

    Минусы: Для больших объектов (строки, векторы) создание копии — дорого и медленно. И оригинал нельзя изменить.



2. Передача по указателю
Функция получает адрес твоей переменной.

    Как выглядит: void foo(int* ptr)

    Что происходит: Внутри функции создается локальная переменная-указатель ptr, которая хранит адрес твоего оригинала. Теперь у тебя в руках "бумажка с адресом квартиры".

    Как работать: Ты обязан использовать разыменование (*ptr), чтобы читать или изменять значение по этому адресу.

    Результат: Любое изменение *ptr внутри функции напрямую меняет твою оригинальную переменную снаружи.

    При вызове: Ты обязан явно передать адрес: foo(&myVar);. Это напоминание: "Осторожно, функция может изменить мою переменную!"



3. Передача по ссылке
Функция получает псевдоним (ссылку) на твою переменную.

    Как выглядит: void foo(int& ref)

    Что происходит: Никакой копии и даже никакой переменной-адреса (для нас). ref внутри функции — это просто второе имя для той переменной, которую ты передал снаружи.

    Как работать: Работаешь с ref как с обычной переменной. Никакого разыменования! Просто ref = 100;.

    Результат: Любое изменение ref внутри функции напрямую меняет оригинальную переменную снаружи.

    При вызове: Синтаксис чистый, без &: foo(myVar);. Смотря на вызов, не скажешь, что myVar может измениться (в этом минус и плюс одновременно).

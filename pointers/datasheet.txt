Шпаргалка: Указатели и Ссылки (C++)

Представь себе память компьютера как огромный многоквартирный дом. В этом доме живут переменные.
1. Адрес (&)

У каждой квартиры (переменной) есть свой уникальный номер — адрес.

    Как получить? Используй оператор & (амперсанд) перед именем переменной.

    Назначение: Узнать, где именно в памяти «живет» переменная.

cpp

int x = 10;
// &x — это АДРЕС квартиры, где хранится число 10.
// Выражение &x вернет что-то вроде 0x7ffd1ca6e99c

Указатель — это переменная специального типа, которая хранит адрес данных в оперативной памяти.

RSP (Stack Pointer) — это 64-битный регистр указателя стека в архитектуре x86-64. Он аппаратно хранит адрес вершины текущего кадра стека. В C++ с его помощью выделяется память под локальные переменные, передаются аргументы и сохраняются адреса возврата при вызовах функций.


size_t — это базовый беззнаковый целочисленный тип, размер которого в байтах всегда совпадает с размером указателя в вашей системе (например, 4 байта для 32-битных архитектур и 8 байт для 64-битных). Он используется для безопасного хранения размеров объектов, индексов массивов и счетчиков циклов

Сначала давайте научимся создавать указатели на обычные переменные. Это проще всего. У каждой переменной в программе есть свой адрес в оперативной памяти. В языках С и С++ есть специальный оператор &, который позволяет узнать адрес любой переменной.

Пусть у нас в программе объявлена переменная x.

int x = 123;  // переменная x целого типа

                      
                  

Для того чтобы получить адрес переменной, нужно использовать оператор '&'.

cout << &x;  // вывод на экран адреса переменной x


Прежде чем использовать указатель, его нужно создать, то есть объявить как переменную специального типа.

Объявить указатель не сложно:

int*  p1;  // Указатель на целое число

                      
                  

Здесь int* - это тип указателя (указатель на целое значение), а p1 - название указателя.

Тип указателя всегда состоит из типа данных на которые он указывает и символа '*'.

Названия указателей часто начинаются с буквы p (от англ. "pointer" — указатель), хотя это вовсе не обязательно.

Объявим еще несколько указателей на разные типы данных:

float*  p2;      // Указатель на вещественное число

char*   p3;      // Указатель на символ (char)

Cat*    p_cat;   // Указатель на кошку (объект пользовательского типа)

                      
                  

Указатель типа int* может хранить адрес любой переменной типа int, но в него нельзя записывать адреса переменных других типов, например,float или char. Аналогичное правило работает для указателей любых других типо

NULLPTR 

Да, по сути так и есть. int* p1 = nullptr; — это указатель, который ни на что не указывает.

Вот ключевые моменты для понимания:

    Это не «указывает в никуда»
    Указатель не содержит адрес какой-либо переменной или объекта в памяти. Он находится в безопасном, «обнулённом» состоянии.

    Чему он равен?
    Его значение — это специальная нулевая константа. В отличие от старого NULL (который часто был просто числом 0), nullptr имеет собственный тип std::nullptr_t, что делает код безопаснее.

    Главная цель
    Показать: «здесь нет объекта, с этим указателем сейчас работать нельзя»

#include <iostream>

int main()
{
int* p1 = nullptr;
std::cout << p1;


return 0;
}
    

Здесь указатель p указывает на переменную x. Теперь программист может получить доступ к значению переменной (число 123) двумя способами:  через имя переменной (x) и через указатель на нее (p). Для того чтобы обратиться к значению, на которое указывает указатель p, нужно перед ним поставить оператор «звездочка» (*).

cout << *p;  // вывод значения по указателю p

                      
                  

Получим число 123.

Также используя оператор * можно изменить значение по указателю p:

*p = 555;   // изменяем значение по указателю на 555
cout << x;  // будет напечатано 555 (новое значение x)

Давайте рассмотрим функцию, которая получает указатель на целое число и увеличивает его значение на 1.

void Increment(int* p) {
  *p = *p + 1;
}

                      
                  

При вызове этой функции в неё нужно передать указатель на переменную или её адрес:

int x = 1;         
Increment(&x);   // Передаем в функцию адрес переменной x
cout << x;       // Будет выведено число 2

                      
                  

Передача указателей в функцию для того, чтобы функция могла изменить значение переменной (объекта), очень часто встречается на практике.


Динамические переменные

Указатели позволяют управлять оперативной памятью: создавать в ней новые «переменные» и уничтожать их, когда они больше не нужны. Давайте разберемся, как это правильно делать!

Для создания динамической переменной используется специальный оператор new ("new" по-английски "новый"). Он выделяет область памяти для хранения значения заданного типа и возвращает ее адрес. Этот адрес мы сохраняем в указатель, например, p. 

int* p = new int;  // создаем новую динамическую "переменную" целого типа

                      
                  

У такой «переменной» нет имени. Поэтому работать с ней можно только через указатель p. Чтобы получить доступ к значению такой «переменной», используется оператор разыменования: *p — это значение динамической переменной.

*p = 100;    // Присвоим в "переменную" число 100
 
cout << *p;  // Выведем значение "переменной" на экран

                      
                  

Если переменная больше не нужна, то ее нужно «удалить» с помощью оператора delete. Оператор delete освобождает выделенную ранее память.

delete p;

                      
                  

После этого обращаться к этой области памяти нельзя! Это приведет к некорректной работе программы.


МАССИВЫ

В этом примере каждый элемент массива – это значение типа int, которое занимает 4 байта оперативной памяти. Элементы массива другого типа будут занимать другое количество байт.

Узнать размер, занимаемый любым типом данных, всегда можно с помощью оператора sizeof. Выясним, например, сколько байт памяти занимает тип int:

cout << sizeof(int);  // обычно это 4 байта


Указатель а (имя массива) запрещено изменять и, кроме того,  он содержит информацию о размере массива. Поэтому размер всего массива можно узнать с помощью оператора sizeof:

cout << sizeof(a);  // 16 байт

                      
                  

Обычный указатель p можно изменять и в нем нет информации о размере массива. Например, можно сделать так, чтобы указатель p указывал не на начало массива a, а на его первый элемент:

p = &a[1]; // p содержит адрес 1-го элемента массива a

                      
                  

Или даже обнулить (чтобы он никуда не указывал):

p = nullptr; // указатель никуда не указывает

                      
                  

При этом аналогичные операции с указателем a запрещены! 




СТРОКИ
В конце каждой строки обязательно есть символ '\0' — это признак окончания строки. Создать такие строки в С++ можно несколькими способами:

const char* str1 = "HELLO!";
char str2[] = "HELLO!";

                      
                  

При этом терминальный ноль добавляется в них автоматически. С такими строками можно работать как с массивами, например, можно вывести на экран символ с индексом 1:

cout << str1[1];  // Буква 'E'

                      
                  

Однако, строка str1 - не может быть изменена, а строка str2 - может.

str1[1] = 'e';  // Ошибка!
str2[1] = 'e';  // OK

                      
                  

Почему так? Дело в том, что строка str1 создается в области памяти «только для чтения», попытка изменить такую строку приводит к неопределенному поведению, поэтому перед указателем на такие строки добавляют ключевое слово const, которое защищает нас от случайного изменения такой строки. Строка str2 создается в стеке, это просто массив символов и с ней можно делать всё то же самое, что и с обычным массивом



Зачем нужны неизменяемые строки?

Они просто хранят в памяти тексты каких-то сообщений, которые выдает программа, или другие нужные данные. Их изменение запрещено, потому что такие строки помещаются в специальной, защищенной от изменений области памяти программы. Вы просто должны об этом помнить.
Как создать изменяемую Си-строку?

Для этого нужно объявить строку как массив символов.

char str[10] = "FOO";  // Эту строку можно изменять
str[0] = 'B';          // Получится строка 'BOO'

                      
                  

Фактически здесь создается массив из 10 символов, в него записывается строка «FOO» и затем изменяется самый левый символ. В этом случае терминальный ноль также добавляется 

Динамическое выделение памяти под Си-строку

Самый сложный вариант — это динамическое выделение памяти под строку. Он может потребоваться для хранения очень больших строк, которые не помещаются в стеке. 

Пример динамического выделения памяти под строку.

// Выделение памяти под строку
char* str = new char[10];

// Изменение отдельных символов
str[0] = 'F';                
str[1] = 'O';
str[2] = 'O';

// Добавление терминального нуля!
str[3] = '\0';

                      
                  

В этом примере строка «FOO» буквально создается по отдельным символам, и в конце добавляется символ терминального нуля. 

На практике для того, чтобы скопировать в С-строку строковый литерал, используется функция strncpy из библиотеки сstring. У функции strncpy 3 параметра: куда скопировать, откуда или что скопировать и ограничение на количество копируемых символов. Ограничение нужно для того, чтобы не выйти за пределы выделенной памяти, так как это приведет к неопределенному поведению.

#include <iostream>
#include <cstring>

int main() {
    char* str = new char[10];

    // Скопировать в str строку "Foo",
    // но не больше 10 символов.
    strncpy(str, "FOO", 10);       
    
    std::cout << str;
    delete[] str;
}

                      
                  

В библиотеке cstring есть много специальных функций для работы со строками в стиле языка Си. Все они получают в качестве аргумента или возвращают указатели. Но это тема для отдельного урока или даже небольшого курса, поэтому здесь мы не будем их рассматривать.

Как вывести на экран содержимое Си-строки?

Если мы хотим вывести на экран всю строку str, то нужно просто отправить ее в поток std::cout

const char* str = "Hello world!";
cout << str;

А вот если мы хотим вывести отдельные символы строки через пробел, то придется перебрать их в цикле. Это можно сделать с помощью вспомогательного указателя p.

#include <iostream>
using namespace std;

int main() {
  const char* str = "Hello world!";
	
  const char* p = str;    // указатель p на начало строки str

  while(*p != '\0') {     // пока не дойдем до конца строки
    cout << *p << " ";    // печатаем символ по указателю p
    ++p;                  // сдвигаем указатель правее
  }
}

                      
                  

Здесь создается вспомогательный указатель p на начало строки str. Выражение *p - это значение по указателю, то есть символ на который он указывает. Пока указатель не дойдет до символа '\0', печатается текущий символ и указатель p сдвигается правее на 1 байт.


Константность и указатели в C++ *

* Эта тема не изучается в школе, но её знание может потребоваться на зачёте и экзамене по программированию в вузе. Школьники могут её пропустить и сразу перейти к разделу 7.1 Оператор "стрелка".

В программах с указателями часто используется ключевое слово const . Оно может защищать от изменений как данные, так и сам указатель. Пусть в программе объявлена строка:

char data[] = "Cat";

                      
                  

Тогда можно объявить 3 различных варианта указателя:

    const char* str = data; — это указатель на константные данные,
    char* const str = data; — это константный указатель,
    const char* const str = data; — это константный указатель на константные данные.

Давайте разберём, что значит каждая из этих записей.
1. Защита данных (const char* str)

Если const записать слева от *, то защищаются данные, на которые указывает указатель.

const char* str = data; // Указатель на КОНСТАНТНЫЕ ДАННЫЕ

                      
                  

Используя str, нельзя изменять символы, на которые он указывает. Компилятор выдаст ошибку:

*str = 'B';    // Ошибка компиляции!
str[2] = 'r';  // Ошибка компиляции!

                      
                  

Если функция принимает такой указатель, это гарантия, что она не будет изменять ваши данные.
2. Защита указателя (char* const str)

Если const записать справа от *, то есть возле указателя, то защищается сам указатель. Его нельзя перенаправить.

char* const str = data; // КОНСТАНТНЫЙ УКАЗАТЕЛЬ

                      
                  

Можно изменять данные:

*str = 'B';
str[2] = 'r';

                      
                  

Нельзя изменять значение указателя (заставлять его указывать куда-то ещё):

str = nullptr; // Ошибка компиляции!
++str;         // Ошибка компиляции!

                      
                  

3. Двойная защита (const char* const str)

Если объявить указатель так, то нельзя будет изменить ни данные, ни сам указатель.

const char* const str = data; // Ничего не двигается и не меняется

                      
                  

🎯 Полезное правило

    const слева от *  защищает данные.

    const справа от *  (const возле указателя) защищает указатель.

Всё это верно и для других типов данных, а не только для строк. 

const int* p = &n; // Указатель на константные данные

                      
                  

int* const p = &n; // Константный указатель
